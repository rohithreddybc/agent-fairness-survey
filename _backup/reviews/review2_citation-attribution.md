### Detailed Findings by Finding Severity

**MUST-FIX (3 issues):**

**1. blankenstein2025biasbusters cited for demographic tool-selection claims (§1.2, §3.1, §4.3 TID)**

Location: §1.2 ("a model that is near-parity on a QA bias benchmark can still exhibit systematic tool-selection bias that no answer-level metric detects \cite{blankenstein2025biasbusters}"), §3.1 (GENERATE-operator claim), §4.3 (TID "existence evidence... does swapping a subject's demographics change which API the agent calls").

Problem: BiasBusters explicitly states it "does not involve or target any sensitive attributes such as gender, race, nationality, or skin color." The paper studies marketplace provider-preference bias and ordering effects among functionally equivalent APIs — it has no demographic dimension whatsoever. The manuscript uses it throughout to motivate a claim about demographic-conditioned tool routing (whether swapping a decision subject's demographics changes which tool is invoked). These are entirely different fairness constructs. The TID definition in §4.3 is particularly problematic: it frames blankenstein2025biasbusters as "existence evidence" for the counterfactual question "does swapping a subject's demographics change which API the agent calls?" — but BiasBusters never poses that question.

Fix: Retain blankenstein2025biasbusters for the specific, accurate claim that LLM selection policies exhibit systematic preferences independent of task fitness (provider bias, ordering bias). Remove it as evidence for demographic-conditioned tool routing. For the demographic claim, the appropriate citation is xu2026ducx (which does involve demographic groups in a medical context). Add a sentence at §3.1 explicitly distinguishing the two types of tool-selection bias: (a) provider/ordering preference bias (blankenstein) and (b) demographic-conditioned routing (xu2026ducx, with a note that no direct demographic-swap measurement of tool selection exists). This distinction is actually important to the survey's argument and would strengthen §3.1's "What is unmeasured" paragraph.

**2. xu2026ducx description as "model, data, tool-invocation stage" decomposition (§3.0.2)**

Location: §3.0.2, Definition D2 paragraph: "Δ_c is the quantity that the clinical decomposition study of xu2026ducx estimates when it attributes separable shares of a tool-using agent's total unfairness to the model, the data, and the tool-invocation stage."

Problem: DUCX decomposes into (1) tool exposure bias, (2) tool transition bias, (3) model reasoning bias. There is no "data" as a separate component. The "model, the data, and the tool-invocation stage" description is a fabricated tripartite that does not correspond to DUCX's framework. This description sits in the formal definition section (D2) where precision is most critical.

Fix: Replace with "tool exposure, tool transition, and model reasoning stages" — the actual DUCX decomposition terms.

**3. hall2025guiding mechanism described as in-trace step-scorer (§3.4, §5.6)**

Location: §5.6: "hall2025guiding inserts fairness reward models into the planning trace, scoring intermediate reasoning steps against demographic parity criteria." §3.4 (Table 4): listed as acting at "C4 planning trace."

Problem: The paper explicitly states it "leaves the model's internal reasoning untouched and intervenes only in the aggregation stage." It scores complete reasoning chains (not individual steps) after generation is complete. The fairness criteria are equalized opportunity and equalized odds, not "demographic parity." The claim that it "inserts" into the planning trace is mechanistically wrong, which matters because the BCF's P4 requirement is precisely that interventions target the entry locus — and characterizing a post-hoc aggregation method as a planning-trace insertion misrepresents the intervention's position relative to P4.

Fix: Correct to "hall2025guiding trains a Fairness Reward Model that scores complete reasoning chains and reweights them during aggregation, reducing selection of biased reasoning paths; the FRM operates post-generation at the aggregation stage, not within the planning trace itself. Fairness criteria are equalized opportunity and equalized odds." Update Table 3 and Table 4 accordingly to show this as a post-generation aggregation-stage intervention rather than a planning-trace guardrail.
