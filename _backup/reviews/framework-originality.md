(b) STRENGTHENED FRAMEWORK SPEC — "the Bias Conduction Framework (BCF)" (ready to write into §3)

Two-axis model: Axis 1 = ENTRY LOCUS (the existing six components, unchanged). Axis 2 = CONDUCTION OPERATOR — how a disparity introduced at one locus is transformed as it crosses to the next pipeline stage. Every corpus paper becomes a (locus, operator) coordinate; every metric and mitigation is keyed to a coordinate.

Formal core (new §3.0, ~2 pages):

D1 (Agent loop). An agent is a tuple ⟨π, T, M, R⟩ over steps t = 1..T: state s_t = (h_t, m_t) (history, memory); input x contains protected attribute A; action a_t = π(s_t, x) drawn from {tool calls, retrievals, delegations, plan steps, escalations, final decision y}. A trajectory is τ(x) = (a_1, …, a_T, y).

D2 (Component counterfactual disparity). For component c, Δ_c(x) = E[ d( τ(x)↾c , τ(x_{A←a′})↾c ) ] — the expected divergence of the component's action variable under a demographic swap, holding upstream realizations fixed (trace replay). This is the formal object that CFR/MASD (§4.2), tool-invocation disparity, escalation disparity, and plan-allocation disparity (§4.3) all instantiate — supply the canonical formulas §4.3 currently says don't exist.

D3 (Conduction operators). Each pipeline edge c→c′ applies an operator φ with four modes:
- ATTENUATE (φ<1): downstream processing damps the disparity (e.g., debate with identity anonymization — choi2025identity).
- PRESERVE (φ≈1): disparity carried unchanged (memory write of an inferred attribute — staab2023memorization → fan2024fairmt).
- AMPLIFY (φ>1): disparity grows across the edge (multi-agent interaction — nguyen2025social; sycophantic drift — liu2025truth).
- GENERATE (φ acts on zero input): disparity emerges where none entered (emergent multi-agent bias — madigan2025emergent; reasoning-introduced bias — wu2025reasoning).

Conduction equation (first-order): Δ_τ ≈ Σ_c Δ_c · Π_{e ∈ path(c→y)} φ_e, with feedback edges (memory write-back, self-training) giving the recurrence Δ^(t+1) = φ_fb · Δ^(t) + Δ_entry — drift is geometric when φ_fb > 1 (this FORMALIZES §3.6 and wyllie2024fairness/xu2024adapting instead of citing them).

Five propositions the framework yields (these are the "derived insights" CSUR wants):
- P1 (Locality). Every Δ_c is separately measurable by component-level counterfactual intervention with trace replay; bias attribution is well-posed per locus. (Grounded by xu2026ducx's stage decomposition.)
- P2 (Masking). Endpoint token-level parity places no bound on Δ_τ at the action level, because attenuation at the actuation surface can co-exist with amplification upstream. (Explains li2025actions; this IS the measurement gap, now as a theorem-shaped claim.)
- P3 (Super-additivity / the agentic multiplier). Δ_τ exceeds Σ_c Δ_c iff the path contains at least one AMPLIFY or GENERATE edge — i.e., a state-writing or inter-agent edge. Predicts: architectures with memory write-back or delegation compound; stateless tool pipelines do not. (This is exactly H2 of §6, now derived rather than asserted.)
- P4 (Mitigation matching). An intervention reduces Δ_τ only if it acts at or upstream of the entry locus AND on the active operator. Predicts the §5 finding (token-level debiasing fails for tool-routing bias) instead of merely observing it; reorganizes Table 3 as (locus × operator) coverage.
- P5 (Measurement adequacy). A benchmark certifies locus c only if it observes c's action variable under counterfactual swap. Corollary: every "Agentic? = no" row of Table 2 certifies nothing about Δ_c for c ≠ final answer — the §4.4 punchline, derived.

Worked example (new §3.8, ~1 page): walk a hiring-screener agent (the §6 C0→C3 ladder) through BCF prospectively. C0: single locus, no edges → Δ_τ = Δ_decision (QA audits suffice). C1: adds tool-selection locus + PRESERVE edge → P2 predicts masking (= H1). C2: adds memory/plan loci + feedback edge (φ_fb) → recurrence predicts intra-trajectory drift. C3: adds delegation locus + GENERATE/AMPLIFY edges → P3 predicts super-additivity (= H2), with choi2025identity's anonymization as the P4-predicted counter-lever. Close with the analysis recipe (4 steps: enumerate loci → classify edges by operator → instrument Δ_c per P1/P5 → match mitigations per P4) any auditor can apply to a new system.
